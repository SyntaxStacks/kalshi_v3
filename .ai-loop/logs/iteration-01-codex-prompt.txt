# Active Tranche Spec

## Goal

Implement tranche 2 of false-negative trade blocker fixes in `kalshi-v3`.

Mission:
- increase valid trade creation and routing
- remove cases where a good crypto trade is skipped or blocked incorrectly
- preserve explicit live safety controls

This tranche is about:
1. recent-decision dedupe
2. snapshot/front-contract selection
3. live size gating using executable price instead of only theoretical price
4. safer lane policy lookup to reduce false `lane_untrained` / readiness misses

## Scope

In scope files and modules:
- `crates/decision/src/main.rs`
- `crates/storage/src/pg.rs`
- small supporting changes in shared helpers only if required by the above
- tests near the touched modules

Exact failure modes to fix:

1. Recent-decision dedupe is still lane-only.
- Different markets in the same lane can still be skipped inside the poll window.
- Deduping must become market-aware:
  - primary key: `(lane_key, market_id)`
  - `market_ticker` fallback only when needed

2. Snapshot/front-contract selection is still not best-tradable-contract aware.
- Regime-aware selection may still pick the wrong market inside a regime.
- Selection must be deterministic and tradability-oriented.
- Minimum acceptable ranking:
  1. inside decision horizon
  2. positive `seconds_to_expiry`
  3. fresher market data
  4. better venue/tradability when available
  5. nearer tradable expiry inside the regime

3. Live size gating still uses theoretical entry price.
- Decision should not reject a live trade as `size_too_small_for_live_contract` if the estimated executable book price would support at least one contract.
- Use available book fields first:
  - `best_bid`
  - `best_ask`
  - side
  - spread / venue quality when useful
- Fall back deterministically when book data is missing.

4. Lane policy lookup is still too brittle.
- Do not rely on fragile SQL text slicing as the only identity check.
- Reduce false negatives in:
  - champion lookup
  - readiness lookup
  - `lane_untrained`
- If full typed schema is too large for this tranche, use a safer validated helper that parses lane identity explicitly and matches:
  - symbol
  - strategy family
  - expiry regime

What must not change:
- do not add new strategies

## Acceptance Criteria

### Recent-decision dedupe
- same lane + same market + same poll window => duplicate skipped
- same lane + different market + same poll window => not skipped

### Snapshot selection
- when two markets exist in the same `(symbol, window, regime)`, the nearer tradable contract wins
- an out-of-horizon market must not win
- a stale-but-newer snapshot must not beat a fresher/tradable candidate incorrectly

### Live size gate
- a trade that supports `>= 1` live contract at estimated executable price is not blocked solely by worse theoretical pricing
- missing book data falls back safely and deterministically

### Lane policy lookup
- valid policy is found for the intended symbol / family / regime
- lookup does not fail because of brittle `split_part(...)`-style parsing
- unrelated or opposite-side rows do not incorrectly win if they should not

### General
- preserve current live safety controls
- `cargo test --workspace` passes
- any new tests for this tranche pass
